## What's different on first launch

Compared to the last build you saw on the old machine, the dev binary is the same shape but a few things changed beneath:

- **MASTER OUT bars actually move now.**  The collapsed-height bug is fixed (b6d87fc).  When you hit play, the L / R bars should jump up to wherever your peaks are.  A green "LIVE" pill appears in the panel header during playback — if it's missing, playback isn't actually running.
- **MASTER OUT bars are now LUFS, not peak.**  Live BS.1770 momentary loudness (400 ms K-weighted window) drives the bars.  The headline number switches between "Momentary LUFS" (during playback) and "Integrated LUFS" (when paused, last analyzed value).  The True-Peak bar on the right of the meter still tracks live peak.
- **All Advanced sliders are drag-to-engage.**  No more "Set" button.  Drag any slider to engage it; clear the numeric field or double-click the slider to reset to Auto.  AUTO pills next to each label show state at a glance.
- **Input / Output gain have their own knob-less slider (GainField).**  Always-on, 0 dB neutral, double-click to reset.
- **Save As / Open Project.**  Two new icon tiles in the top header next to settings/help: folder (Open) and disk (Save As).  Files are `.ams.json`.
- **Signal chain strip** above the transport.  Eight stages — Source, EQ, Warmth, Air, Comp, Width, Saturation, Limiter — light up in proportion to their current setting magnitude.  Links between active stages animate a flow gradient.
- **Tape preset is quieter, Spatial is louder and wider** by default.  Re-listen to compare.

## How to spin up the dev window

```powershell
# from the repo root
npm install                  # if node_modules is missing
npm run tauri dev            # opens the desktop window
```

The Rust binary takes ~15-30 s to compile cold.  Vite ready takes ~0.4 s.

If you get exit code `0xffffffff` AFTER you closed the window — that's normal.  Tauri's CLI reports a forcibly-terminated child as a non-zero exit.  Just re-run.

If you want to regenerate the project-save smoke fixture:

```powershell
cd src-tauri
cargo run --bin produce_dialog_smoke
```

That writes `test-output/tauri-project-dialogs-smoke/native-dialog-save-as.ams.json` (3 KB, real `.ams.json` shape).  The file is also committed at HEAD so a fresh Claude session can read it without running the binary.

### 4.  Compression density listening

Set Intensity to ~70 %, pick a track with clear transients, and sweep Compression Density 0 → 1.  Watch the per-band GR pills in the bottom status bar (L / M / H).  Expectations:

- Mid band fires first on most material (vocals / snare).
- Low band has slower attack (30 ms vs 15 ms mid vs 5 ms high) so it doesn't pump on kicks.
- At density = 0.5, threshold is at -12 dBFS uniformly.
- At density = 1.0, threshold is at -24 dBFS uniformly.

If the compressor sound is wrong, the per-band defaults live in `dsp.rs` near `LOW_RATIO_DEFAULT` etc.  Per-band overrides are exposed in the Advanced panel's "Per-band compressor" collapsible.
