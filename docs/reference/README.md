# Reference Docs

These files are optional context from the Codex/current-repo path.

They are useful for understanding decisions already made, but they should not force this repo to copy the current implementation.

Files:

- `codex-implementation-plan.md`: the current Codex repo execution plan.
- `codex-research-implementation-notes.md`: notes on research already applied in the current repo.

Use these when helpful. Prefer `docs/PRODUCT.md` and `docs/CLAUDE_BUILD_BRIEF.md` when making decisions for this clean build.
